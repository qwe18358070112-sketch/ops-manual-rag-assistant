# 共享部署与迁移手册

更新时间：2026-04-20

## 适用目标

这份手册解决两个问题：

1. 让同一局域网或服务器上的其他人也可以使用政企运维知识库问答助手。
2. 让项目迁移到另一台电脑或服务器时，不需要手工拆分复制“本地文件 + 本地数据库 + 本地索引”。

## 一、当前数据存储方式

系统当前采用三层本地存储：

- 原始资料文件：`$OPS_ASSISTANT_DATA_DIR/raw/uploads/`
- 资料元数据 SQLite：`$OPS_ASSISTANT_DATA_DIR/library/metadata.sqlite3`
- 检索索引与中间产物：
  - `extracted/`
  - `chunks/`
  - `indexes/`

默认情况下，`OPS_ASSISTANT_DATA_DIR` 为空时，数据目录就是项目内 `data/`。

## 二、推荐共享部署方式

### 方案 A：局域网共享

适用于你把服务跑在自己电脑上，让同事在同一网络里访问。

`.env` 建议配置：

```bash
OPS_ASSISTANT_ENV=shared
OPS_ASSISTANT_DATA_DIR=/home/lenovo/projects/ops-manual-rag-assistant/data
OPS_ASSISTANT_HOST=0.0.0.0
OPS_ASSISTANT_PORT=8000
OPS_ASSISTANT_REQUIRE_AUTH=true
OPS_ASSISTANT_ADMIN_USERNAME=admin
OPS_ASSISTANT_ADMIN_PASSWORD=change-me
```

启动：

```bash
./scripts/start_shared.sh
```

访问方式：

```text
http://你的电脑IP:8000/app
```

建议：

- 普通使用人员只用 `/app/search`、`/app/timeline`、`/app/topics`
- `/app/library` 保持管理员认证开启

### 方案 B：服务器 / 小主机部署

适用于后期迁到办公室服务器、云主机或小型工控机。

推荐：

- 把代码放在应用目录
- 把 `OPS_ASSISTANT_DATA_DIR` 放到单独磁盘或挂载目录
- 用 `systemd` 或 `docker compose` 启动

这样后续升级代码时，数据目录不需要跟着挪。

## 三、迁移到其他电脑或服务器

### 方式 1：迁移包 ZIP

这是最推荐的迁移方式。

导出：

```bash
.venv/bin/python scripts/manage_library.py bundle-export --output /tmp/ops-assistant-bundle.zip
```

导入：

```bash
.venv/bin/python scripts/manage_library.py bundle-import --input /tmp/ops-assistant-bundle.zip
```

覆盖导入：

```bash
.venv/bin/python scripts/manage_library.py bundle-import --input /tmp/ops-assistant-bundle.zip --replace
```

特点：

- 会带上原始资料文件
- 会带上资料标题、标签、日期、备注、分类
- 导入后自动执行重建索引
- 不需要手工拷贝多个目录

### 方式 2：直接迁移外置数据目录

如果你已经把 `OPS_ASSISTANT_DATA_DIR` 放在独立目录，也可以直接搬整个目录：

```text
$OPS_ASSISTANT_DATA_DIR/
```

这种方式适合：

- 同一系统内换盘
- 同构服务器快速迁移

不适合：

- 只想迁移部分资料
- 想保留旧环境再做并行验证

## 四、systemd 方式

参考：

- `deploy/systemd/ops-manual-rag-assistant.service`

要点：

- `EnvironmentFile=.env`
- `ExecStart` 调用 `scripts/start_shared.sh`
- `OPS_ASSISTANT_DATA_DIR` 可指向外部磁盘

## 五、Docker 方式

参考：

- `Dockerfile`
- `deploy/docker-compose.yml`

启动：

```bash
cd deploy
docker compose up -d --build
```

当前 compose 已把数据目录挂成 volume，更适合长期部署。

## 六、当前建议

如果你短期内还在自己电脑上用：

- 直接用 `start_shared.sh`
- 开启管理员认证
- 定期导出迁移包 ZIP 备份

如果你准备给别人长期使用：

- 用外置数据目录
- 用 `systemd` 或 `docker compose`
- 把迁移包导出当成日常备份手段
