# 政企运维知识操作手册

## 1. 手册说明

本手册面向两类使用者：

- 项目维护者：负责文档接入、索引构建、系统运行、问题排查
- 演示或使用者：负责查询手册、检索历史案例、做专题复盘

本系统名称为：`政企运维知识库问答助手`

它的核心目标不是替代运维人员，而是提升以下两件事的效率：

- 快速定位操作手册中的标准步骤
- 快速复盘周报月报中的真实处理记录

---

## 2. 系统能力概览

当前系统已支持：

- 导入 `docx/pdf/xlsx/txt`
- 按章节、步骤、表格行进行结构化切块
- 关键词 + 语义混合检索
- 引用式答案生成
- 手册问答 / 历史案例检索双模式
- `system_name/source_type/tag/date` 多维过滤
- section 级标签
- 历史案例时间轴
- 专题总览、专题详情、案例详情
- 案例到问答联动
- 问答引用反向跳转到案例详情

## 3. 知识源范围

当前知识源主要分为两类：

### 3.1 手册类

- 综治中心设备维护和应急处理工作手册
- 指挥大厅和视频平台应急处理预案
- 视频平台巡检流程和操作手册
- 视联网会议操作手册
- 视频融合赋能平台官方手册
- 平安风险智控操作手册

### 3.2 案例类

- 2026 年 1 到 3 月月报
- 2026 年 2 到 4 月周报

作用区分：

- 手册类用于回答“怎么操作”“步骤在哪”“先查什么”
- 案例类用于回答“以前怎么处理过”“某个故障有哪些历史记录”

## 6. 系统启动流程

### 6.1 导入文档

```bash
python3 scripts/ingest_documents.py
```

执行后会生成：

- `data/extracted/*.json`
- `data/chunks/*.json`
- `data/indexes/ingestion_summary.json`

### 6.2 构建索引

```bash
python3 scripts/build_retrieval_indexes.py
```

执行后会生成：

- `data/indexes/retrieval.sqlite3`
- `data/indexes/semantic_index.json`
- `data/indexes/retrieval_summary.json`

### 6.3 启动服务

```bash
.venv/bin/uvicorn app.main:app --reload
```

启动后可访问：

- `/`
- `/healthz`
- `/catalog`
- `/search-page`
- `/case-timeline-page`
- `/topic-view-page`

## 7. 页面功能说明

## 7.1 首页

入口：

- `/`

作用：

- 项目总入口
- 可直接进入问答页
- 可跳转到案例时间轴和专题视图

---

## 7.2 问答页

入口：

- `/search-page`

适用场景：

- 查操作步骤
- 查某类故障历史处理方式
- 对比手册和案例中的差异

支持参数：

- `q`
- `search_mode`
- `system_name`
- `source_type`
- `tag`
- `date_from`
- `date_to`
- `rewrite`

`search_mode` 说明：

- `manual_qa`：只查手册
- `case_search`：只查案例
- `all`：全量混合检索

返回内容：

- 简短答案
- 置信度
- 引用来源
- 检索片段

使用建议：

- 查步骤，用 `manual_qa`
- 查历史故障，用 `case_search`
- 不确定时，用 `all`

---

## 7.3 历史案例时间轴

入口：

- `/case-timeline-page`

作用：

- 按时间顺序查看历史案例
- 支持按专题标签、时间范围、来源类型筛选

适合场景：

- 查看某一时间段发生了哪些问题
- 复盘某段时间的驻场工作
- 找某个专题的时间演进过程

每条案例支持：

- 查看摘要
- 查看标签
- 查看案例详情
- 联动到问答

---

## 7.4 专题总览页

入口：

- `/topic-view-page`

作用：

- 按 section 级标签查看高频专题

典型专题：

- 设备巡检
- 轮询助手
- 漏洞整改
- 客户端登录
- 网络故障
- 大屏处理

每个专题支持：

- 查看样例案例
- 进入专题详情
- 联动到问答

---

## 7.5 专题详情页

入口示例：

```text
/topic-detail-page?tag=轮询助手
```

作用：

- 查看某个专题下的完整案例集合

适合场景：

- 单独展示轮询助手相关工作
- 单独展示漏洞整改专题
- 单独展示客户端登录专题

每条案例支持：

- 查看案例详情
- 联动到问答

---

## 7.6 案例详情页

入口示例：

```text
/case-detail-page?section_id=ops-case-010-tabular-0003
```

作用：

- 查看某一条案例的完整原文
- 查看该条案例的来源文件、标签、时间、资料类型

适合场景：

- 面试时展示“我不是只会搜，我能回到原始案例”
- 复盘某条故障记录
- 从问答引用反向回源

## 8. 常用演示路径

### 8.1 手册问答演示

1. 打开 `/search-page`
2. 输入 `视频平台资源共享怎么操作`
3. 选择 `manual_qa`
4. 展示答案、引用、原始片段

### 8.2 案例检索演示

1. 打开 `/search-page`
2. 输入 `视频平台客户端登录不上怎么处理`
3. 选择 `case_search`
4. 选择标签 `客户端登录`
5. 展示真实案例

1. 打开 `/case-timeline-page?tag=轮询助手`
2. 展示轮询助手相关案例时间顺序
3. 点击 `联动到问答`
4. 展示自动带入的问答页

1. 打开 `/topic-view-page`
2. 点击 `轮询助手`

4. 再点击某条 `案例详情`
5. 再从详情页点击 `联动到问答`

## 9. 过滤项说明

### 9.1 system_name

表示知识来源所属系统，例如：

- 视频平台
- 视联网
- 综治中心
- 平安风险智控
- 政企驻场运维

### 9.2 source_type

表示资料类型：

- `internal_manual`
- `official_manual`
- `emergency_plan`
- `weekly_report`
- `monthly_report`

### 9.3 tag

表示 chunk/section 级标签，例如：

- 轮询助手
- 漏洞整改
- 客户端登录
- 网络故障
- 资源共享
- 大屏处理
- 设备巡检

### 9.4 date_from / date_to

用于案例类过滤，适合查看某一时间段的问题记录。

## 11. 常见问题排查

### 11.1 搜不到结果

先检查：

- 是否选错 `search_mode`
- 是否过滤条件过多
- 标签是否选得太细
- 日期范围是否过窄

建议：

- 先用 `all`
- 先去掉 `tag`
- 缩短提问句子

### 11.2 结果不够准

先检查：

- 该问题更像“手册问题”还是“案例问题”
- 是否需要指定系统名
- 是否需要限定时间范围

建议：

- 操作步骤类问题优先 `manual_qa`
- 历史处理类问题优先 `case_search`

### 11.3 标签不够细

可调整：

- `app/ingestion/tagging.py`

这里维护了 section 级标签规则。

### 11.4 引用跳转不到案例详情

先检查：

- 引用是否来自 `weekly_report/monthly_report`
- `section_id` 是否存在于案例库中

说明：

- 目前只有案例类引用支持反向跳转
- 手册类引用的详情页后续再补

## 12. 维护建议

### 12.1 日常维护

- 新增一批周报/月报后，及时重跑导入与索引
- 每次加入新资料后，抽样验证 5 到 10 个问题

### 12.2 演示维护

- 固定一套演示问题，不要现场随机发挥
- 每次演示前检查：
  - 服务已启动
  - 时间轴页可打开
  - 专题详情页可打开
  - 问答引用跳转可打开

### 12.3 面试维护

- 保持项目描述统一
- 强调“真实政企运维数据 + RAG + 引用式答案”
- 不要夸大成“自动诊断平台”
